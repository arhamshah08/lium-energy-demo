Developer — Publish for Finance
Makes project visible to all financiers on the discovery feed
Developer POST /api/projects/{id}/publish → PUBLISHED_FOR_FINANCE ▼
Request body
{
  "target": "finance"
}

// target options: "finance" | "sa"
// "sa" requires status=FINANCING_ACCEPTED
//       AND ptoStatus=APPROVED
Response 200
{
  "ok": true,
  "data": {
    "id": "proj_abc123",
    "status": "PUBLISHED_FOR_FINANCE",
    "ptoStatus": "PRE_PROCESSING",
    /* full Project object */
  }
}
// Project now visible on /projects
// for all financiers (auto-polls every 5s)
Guard: If project is not yet SUBMITTED, the endpoint auto-verifies telemetry so developers can publish in a single click during demos.

4
Developer — Accept / Reject / Request Revision
All actions via PATCH on the individual offer
Developer PATCH /api/projects/{id}/offers/{offerId} → FINANCING_ACCEPTED ▼
Accept offer
{
  "action": "accept"
}

// Effects:
// • This offer → ACCEPTED
// • All other PENDING/REVISION_REQUESTED
//   offers → REJECTED (auto)
// • Project status → FINANCING_ACCEPTED
Reject offer
{
  "action": "reject"
}

// Only this offer → REJECTED
// Other offers remain open
// Project stays at OFFER_RECEIVED
Request revision
{
  "action": "request_revision",
  "revisionNotes": "Please reduce spread to 2.25% and extend tenor to 20 years"
}

// This offer → REVISION_REQUESTED
// Financier sees the note in My Offers
// Project stays at OFFER_RECEIVED
Response for all actions 200
{
  "ok": true,
  "data": {
    "id": "offer_xyz789",
    "status": "ACCEPTED",  // or REJECTED / REVISION_REQUESTED
    "revisionNotes": null,  // populated if revision
    "updatedAt": "2026-05-19T12:00:00Z"
  }
}

Financier — Resubmit Revised Offer
Triggered after developer requests revision — pre-fills previous terms
Financier PATCH /api/projects/{id}/offers/{offerId} → PENDING again ▼
Request body
{
  "action": "resubmit",
  "updatedTerms": {
    "sofrSpreadPct": 2.25,    // revised down
    "tenorYears": 20,          // extended
    "expiresAt": "2026-06-30T23:59:59Z"
  }
}

// Only the financier who submitted the offer
// can resubmit it. Any subset of terms can
// be updated — unspecified fields retain
// their previous values.
Response 200
{
  "ok": true,
  "data": {
    "id": "offer_xyz789",
    "status": "PENDING",
    "sofrSpreadPct": 2.25,
    "tenorYears": 20,
    "revisionNotes": null,  // cleared on resubmit
    "expiresAt": "2026-06-30T23:59:59Z"
  }
}

Request body

}

Request body
{

  "amountINR": 30
}

// investorType options:
// PENSION_FUND | INSURANCE | CREDIT_FUND

All Actors ALL Error response envelope ▼
Error response shape
{
  "ok": false,
  "error": {
    "code": "INVALID_STATE",
    "message": "Project is not open for financing offers"
  }
}
Common error codes
"UNAUTHORIZED"     // 401 — missing/invalid JWT
"FORBIDDEN"         // 403 — wrong role
"NOT_FOUND"         // 404 — resource not found
"INVALID_STATE"     // 422 — wrong lifecycle status
"PTO_REQUIRED"      // 422 — PTO must be APPROVED
"INVALID_ACTION"    // 400 — unknown action field
"INVALID_TARGET"    // 400 — publish target unknown
